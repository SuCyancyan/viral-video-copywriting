# 短视频爆款文案生产线

四步流水线，从拆解到出品一条爆款文案。

## 路由

| 用户意图 | 子能力 | 核心指引文件 |
|----------|--------|-------------|
| 拆解爆款、分析爆款、拆解大纲 | Skill 1 拆解大纲 | `references/skill1-deconstruct.md` |
| 写大纲、生成大纲 | Skill 2 写大纲 | `references/skill2-outline.md` |
| 写稿、写文案、扩充大纲 | Skill 3 写稿 | `references/skill3-draft.md` |
| 改稿、验证、核查、查证 | Skill 4 改稿验证 | `references/skill4-verify.md` |
| 完整流程（未指定步骤） | 全流程顺序执行 | 按 1→2→3→4 依次路由 |

## 全流程概览

```
用户输入爆款Excel → [Skill 1 拆解大纲] → 结构化拆解
                                         ↓
              用户可选附件 ──→ [Skill 2 写大纲] → 3套大纲方案
                                                    ↓
              爆款标题库 ──→ [Skill 3 写稿] → 完整文案（含来源标注）
                                                  ↓
                             [Skill 4 改稿验证] → 修改建议
                                                  ↓
                                              用户定稿 ✅
```

## 执行原则

1. **逐步执行，不跳步**：即使是一次性全流程，也要按 1→2→3→4 顺序输出，每步完成后等用户确认再进入下一步
2. **权重驱动**：点赞数是所有步骤的核心权重依据，高赞爆款的元素优先级最高
3. **数据可溯源**：从 Skill 3 写稿开始，所有数据、案例必须标注来源链接，Skill 4 逐条验证
4. **用户拍板**：Skill 2 输出3套方案由用户选，Skill 4 只给建议不改稿，最终决定权在用户

## 关键输入格式

Skill 1 需要用户提供 Excel 表，格式要求：

| 列名 | 说明 | 必填 |
|------|------|------|
| 爆款文案原文 | 完整文案文本 | ✅ |
| 点赞数 | 用于权重排序 | ✅ |
| 爆款标题 | 用于封面参考 | ✅ |
| 来源链接 | 用于验证 | ✅ |

用户可通过以下方式提供：
- 直接发送 .xlsx/.csv 文件
- 直接在对话中粘贴表格内容
- 发送图片截图（我会 OCR 识别）

## 通用陷阱

- **不要跳过拆解直接写大纲**：没有拆解的写大纲是无根之木，必须先拆
- **不要忽略权重**：所有爆款平等对待 = 没有对待，必须按点赞加权
- **不要凭空编数据**：写稿时找不到的数据，宁可留空标注"待补充"，也不编造
- **不要过度纠错**：改稿验证时聚焦事实性错误，不纠结措辞偏好
