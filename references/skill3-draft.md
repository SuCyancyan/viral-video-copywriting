# Skill 3：写稿

将大纲扩充为完整文案，确保逻辑连贯、措辞符合爆款风格。

## 输入要求

| 输入项 | 来源 | 必填 |
|--------|------|------|
| 选定大纲 | Skill 2 用户选定方案 | ✅ |
| 爆款标题库 | Skill 1 输入表格中的"爆款标题"列 | ✅ |
| 补充材料 | 用户额外提供的案例、数据等 | ❌ |

## 执行流程

### 第1步：提取流量词

从爆款标题库中提取高频流量词，用于开头和关键点位：

```
提取规则：
  1. 列出所有爆款标题
  2. 提取重复出现的关键词（如"搞钱""信息差""普通人"）
  3. 标注每个词的来源爆款权重

示例：
  - "搞钱"：出现2次，来源🥇#1 + #3 → 高频流量词
  - "信息差"：出现1次，来源🥈#2 → 中频流量词
  - "普通人"：出现2次，来源#1 + #4 → 高频流量词
```

### 第2步：写开头

规范：
1. 严格按大纲开头的元素结构写
2. 将流量词自然嵌入开头句子
3. 每句话必须有明确目的（不能有无意义的废话）
4. 开头总长度控制在全文的 10-15%

开头句子校验：

| 句子 | 目的 | 流量词 | ✅/❌ |
|------|------|--------|------|
| 例句 | 悬念钩子 | "搞钱" | ✅ |
| ... | ... | ... | ... |

如果某句话无法标注目的 → 删掉重写。

### 第3步：写正文

规范：
1. 严格按大纲 Part 展开，不增不减
2. 每个 Part 的结构：小标题 → 核心观点展开 → 论据/案例支撑 → 过渡到下一Part
3. 过渡词要自然，参考爆款原文的过渡方式（如"但问题是…""所以你想啊…"）
4. 口语化：禁止书面化长句，单句不超过25字为佳

有附件 vs 无附件写法区别：

| 维度 | 有附件 | 无附件 |
|------|--------|--------|
| 内容来源 | 严格按附件写 | 参考爆款内容重组 |
| 专业度 | 保留专业术语，用口语解释 | 按爆款风格写 |
| 数据引用 | 必须标注来源 | 引用爆款数据，也需标注 |

### 第4步：数据引用标注

所有涉及数据、案例、网站、论文的内容，必须标注来源：

```
标注格式（类论文引用）：
  [来源序号] 来源标题 - URL

示例：
  "根据麦肯锡2024年报告，AI将取代30%的重复性工作[1]"
  
  参考来源：
  [1] 麦肯锡全球研究院 - The Economic Potential of Generative AI - https://...
```

标注规则：
- 爆款原文提到的数据 → 标注爆款的来源链接
- 附件中的数据 → 标注附件文档名称/页码
- 自主搜索补充的数据 → 标注搜索到的网页URL
- 无法找到可靠来源的数据 → 标注【待核实】并提示用户

### 第5步：补充机制

写稿过程中发现缺少信息时：

```
优先级：
  1. 自主搜索 → 找到可靠来源 → 写入文案并标注
  2. 搜索无果 → 文案中留【待补充：缺少XXX数据/案例】→ 提示用户
  3. 绝不编造 → 任何无法验证的数据一律不写
```

### 第6步：写结尾

按大纲结尾方向写，简洁有力：
- 行动号召型：1-2句引导互动
- 金句总结型：1句升华点题
- 价值预告型：1句预告下期 + 1句引导关注

## 输出模板

```
# 文案标题：[从爆款标题库提炼/重写的标题]

## 流量词入库
| 流量词 | 出现次数 | 来源权重 |
|--------|---------|---------|

---

[开头部分]

[正文]

### Part One：小标题
内容...

### Part Two：小标题
内容...

...

[结尾]

---

## 参考来源
[1] 来源标题 - URL
[2] 来源标题 - URL
...

## 待补充项（如有）
- 【待补充】缺少XXX数据
- 【待核实】XXX案例来源未确认
```

## 注意事项

- 开头每句话的目的校验是硬性要求——没有目的的句子就是流失点
- 流量词嵌入要自然，不能生硬塞入
- 数据标注是 Skill 4 验证的前提，不标注 = 没法验证 = 后续全废
- 口语化不等于低质——参考爆款的表达力，口语也能有信息密度
