# Skill 2：写大纲

基于 Skill 1 的拆解结果，生成新文案大纲。

## 输入要求

| 输入项 | 来源 | 必填 |
|--------|------|------|
| 拆解结果 | Skill 1 输出 | ✅ |
| 用户附件 | 技术文档、参考资料等 | ❌ |
| 创作场景 | 有附件 vs 无附件（影响正文逻辑） | ✅ |

创作场景判定：
- **有附件**：正文 Part 严格按附件内容梳理，爆款结构只管开头和结尾
- **无附件**：正文 Part 按爆款内容排列重组

## 执行流程

### 第1步：开头设计

统计 Skill 1 拆解中各爆款开头元素的重复度，确定必选元素：

```
开头元素统计规则：
- 重复出现 ≥ 2次的元素 → 必须纳入大纲
- 只出现1次但来自🥇爆款的 → 强烈建议纳入
- 只出现1次且非🥇爆款的 → 可选
```

开头结构模板：

| 元素位置 | 元素类型 | 参考来源 | 说明 |
|---------|---------|---------|------|
| 第1句 | 例如：悬念钩子 | 爆款#1开头句1 | 制造信息缺口 |
| 第2句 | 例如：痛点共鸣 | 爆款#2开头句2 | 击中焦虑 |
| 第3句 | 例如：预告价值 | 爆款#1开头句3 | 承诺价值 |
| ... | ... | ... | ... |

### 第2步：正文设计

#### 场景A：有附件

1. 通读附件，提炼核心内容模块
2. 将模块映射到 Part 结构（参考爆款 Part 数量）
3. 每个 Part 对应附件的一个核心模块
4. 用爆款的口语化风格改写附件的书面表达

```
附件内容 → Part 映射规则：
- 附件章节/段落 ≈ 1个Part
- 若段落过多，合并同类项（同一主题的归入同一Part）
- 若只有一个大主题，按"是什么→为什么→怎么做→避坑"拆分
```

#### 场景B：无附件

1. 提取各爆款正文 Part 的核心观点
2. 按主题合并同类项：同一观点出现多次的权重更高
3. 按逻辑排序（通常：认知→方法→避坑→行动）

```
爆款正文合并规则：
- 观点相同或相近 → 合并为一个Part，标注参考了哪几篇
- 观点互补（如一个说方法、一个说避坑） → 并列保留
- 观点冲突 → 优先采纳🥇爆款的版本，标注分歧
```

### 第3步：结尾设计

统计爆款结尾类型占比，选最多的类型：

- 行动号召型（"点赞收藏"/"评论区见"）→ 流量导向
- 金句总结型（升华主题）→ 调性导向
- 价值预告型（下期预告）→ 系列导向

### 第4步：生成3套方案

每套方案需差异化：不能只是换个说法，结构要有区别。

差异化维度：
- **开头策略**：哪个爆款开头为主模板
- **正文深度**：偏干货（多Part细展开）vs 偏观点（少Part强输出）
- **结尾导向**：引流 vs 升华 vs 预告

## 输出模板

```
## 大纲方案 A [差异化标签：如"干货深度型"]

### 开头
| 位置 | 元素类型 | 参考来源 | 要点 |
|------|---------|---------|------|

### 正文
| Part | 小标题 | 核心观点 | 参考来源 |
|------|--------|---------|---------|

### 结尾
| 类型 | 方向 | 参考来源 |
|------|------|---------|

---

## 大纲方案 B [差异化标签：如"观点冲击型"]
...

---

## 大纲方案 C [差异化标签：如"行动导向型"]
...
```

## 注意事项

- 开头元素重复度统计是核心——它决定了大纲的"骨架"是否站得住
- 有附件时，附件内容 > 爆款内容（正文部分），爆款只提供结构和风格
- 无附件时，必须标注每个Part参考了哪些爆款，避免凭空发明
- 3套方案必须在结构上有明显差异，不能只是措辞微调
